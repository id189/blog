# 霓光志 · 项目架构文档

> 技术栈：Astro 4 + Pagefind 1.3 + TypeScript（strict）。纯静态生成，无运行时数据库。
> 本文档与 `blog/` 目录代码保持一致，覆盖项目概述、目录结构、模块说明、数据流程与配置说明。

---

## 1. 项目概述

| 项 | 内容 |
| --- | --- |
| 项目名称 | `ai-neon-blog`（中文名「霓光志」） |
| 用途 | 个人 AI 技术博客：用 Markdown 写文章，推送到 GitHub 即自动构建并发布 |
| 核心框架 | **Astro 4**（静态站点生成器，默认 `output: static`） |
| 站内搜索 | **Pagefind 1.3**（构建期对 `dist/` 建立全文索引，可搜文章正文） |
| 开发语言 | TypeScript（`astro/tsconfigs/strict`），路径别名 `@/* → src/*` |
| UI 依赖 | 无额外 UI 框架；纯 Astro 组件 + 原生 CSS 变量设计系统 |
| 部署方式 | 基于 GitHub 仓库同步：Netlify / Cloudflare Pages / Vercel / **GitHub Pages**（仓库自带 Actions 工作流）四选一 |

### 核心功能

- **纯静态生成**：`npm run build` → `dist/`，无运行时服务器、无数据库，可直接托管到任意静态平台。
- **Markdown 写作**：基于 Astro **Content Collections**。往 `src/content/posts/` 丢一个 `.md` 即自动生成首页列表与 `/posts/<slug>` 详情页，无需手动登记。
- **全文搜索**：首页右侧栏接入 Pagefind，可搜进**文章正文**；开发态未构建时自动降级为「标题/描述/标签/分类」文本匹配。
- **分类筛选**：右侧栏按 `category` 聚合与计数，点击即过滤，且**可与搜索叠加**。
- **文章详情页**：自动目录 TOC（h2/h3） + 滚动高亮（IntersectionObserver）。
- **完整响应式**：桌面 4 列 / 平板 2 列 / 移动 1 列；导航在 ≤720px 折叠为汉堡菜单；侧栏在 ≤1023px 置顶到列表上方。
- **视觉风格**：浅色明快 + 霓虹点缀 —— 玻璃拟态卡片、浮动光球、渐变文字、科技点阵背景。

```

│  │  ├─ Footer.astro        页脚
│  │  ├─ Hero.astro          首页顶部宣传区

│  │  ├─ Toc.astro           文章详情页右侧目录（滚动高亮）
│  │  └─ GlowOrb.astro       装饰性霓虹光球（纯 CSS，无交互）
│  ├─ layouts/
│  │  └─ Base.astro          全局 HTML 骨架：head / 字体 / SEO + Header / Main / Footer 插槽

│  ├─ content/
│  │  ├─ config.ts           Content Collections 的 posts 集合 + frontmatter schema（Zod）

│  │  └─ global.css          设计系统 token + 全局样式 + 全部响应式断点
│  ├─ consts.ts             SITE 站点常量 + NAV 导航配置（改文案 / 站名 / 社交改这里）

│  └─ favicon.svg           站点图标（蓝紫渐变菱形）
```

### 目录职责速查

| 目录 / 文件 | 职责 |
| --- | --- |
| `src/components/` | 可复用的展示/交互组件，被页面或布局引用 |
| `src/layouts/Base.astro` | 全站统一 HTML 外壳，提供 `<slot/>` 供页面填充 |
| `src/pages/` | 路由定义；文件名 / 目录结构即 URL 路径 |
| `src/content/posts/` | 文章「数据源」，新增 `.md` 即新增内容 |
| `src/content/config.ts` | 文章数据契约（schema），决定 frontmatter 校验规则 |
| `src/styles/global.css` | 设计系统唯一来源，全站样式与设计变量 |
| `src/consts.ts` | 站点级文案与导航配置 |
| `public/` | 不经构建、原样拷贝到 `dist/` 的静态资源（图标等） |
| 根配置文件 | 构建 / 类型 / 部署行为定义 |

## 3. 模块说明

### 3.1 布局与组件

| 模块 | 入口文件 | 功能 | 依赖 | 与其它模块交互 |
| --- | --- | --- | --- | --- |
| **Base 布局** | `src/layouts/Base.astro` | 全站 HTML 骨架：`head`（charset / viewport / description / favicon / title）、引入 Google Fonts；`body` 内含 GlowOrb + Header + `<main><slot/></main>` + Footer | Header、Footer、GlowOrb、consts、global.css | 所有页面都 `extends Base`；`title` / `description` 由页面传入，缺省回退 `SITE` |
| **Header** | `src/components/Header.astro` | sticky 顶部导航：brand（logo + 站名）、nav（导航链接，当前页 `active`）、汉堡按钮（≤720px 显示） | consts（SITE、NAV） | 用 `Astro.url.pathname` 判定 active；`<script>` 切换 `#siteHeader` 的 `.nav-open` |
| **Footer** | `src/components/Footer.astro` | 页脚：年份 + 站名 + tagline + Powered by Astro | consts（SITE） | 被 Base 渲染 |
| **Hero** | `src/components/Hero.astro` | 首页宣传区：eyebrow、站名 + tagline 渐变、描述、CTA 按钮 | consts（SITE） | 仅首页 `index.astro` 使用 |
| **PostCard** | `src/components/PostCard.astro` | 列表项卡片：封面（emoji / 图）、日期、标题、摘要、标签（前 3 个）；外层 `<a>` 指向 `/posts/<slug>` | props.post | 由 `index.astro` 在 `posts.map` 中渲染；输出 `data-slug / data-category / data-title / data-desc / data-tags`（小写）供首页脚本过滤 |
| **Toc** | `src/components/Toc.astro` | 文章详情目录：取 `headings` 中 depth 2/3 渲染 `#tocList`；`<script>` 用 IntersectionObserver 高亮当前节 | props.headings | 由 `posts/[slug].astro` 传入 `headings`（来自 `post.render()`） |
| **GlowOrb** | `src/components/GlowOrb.astro` | 3 个装饰光球（blue / violet / cyan），fixed + blur + drift 动画 | 无 | 仅 Base 渲染，纯展示 |

### 3.2 页面（路由）

| 模块 | 入口文件 | 功能 | 依赖 | 与其它模块交互 |
| --- | --- | --- | --- | --- |
| **首页** | `src/pages/index.astro` | 取 `getCollection("posts", !draft)` 按日期降序；统计分类及篇数；渲染 Hero + `home-layout`（网格 + 侧栏）；`<script>` 接 Pagefind 搜索 + 分类过滤 | content、Base、Hero、PostCard、consts | 读取 Content Collections；调用 PostCard；对卡片集合应用 `data-*` 过滤 |
| **文章详情** | `src/pages/posts/[slug].astro` | `getStaticPaths` 为每个 `.md` 生成路由；`post.render()` 得 `Content` + `headings`；`post-layout`（正文 + Toc） | content、Base、Toc | 依赖 Content Collections schema；Toc 消费 headings |
| **关于页** | `src/pages/about.astro` | about-hero（avatar + author + tagline） + about-card（简介 + 社交按钮） | Base、consts | 社交链接取自 `SITE.social` |
| **404** | `src/pages/404.astro` | 简单 404 提示 + 回首页按钮 | Base | — |

### 3.3 数据与配置模块

| 模块 | 入口文件 | 功能 | 依赖 | 与其它模块交互 |
| --- | --- | --- | --- | --- |
| **Content 配置** | `src/content/config.ts` | 定义 `posts` 集合（`type: "content"`）+ Zod schema；新增 `.md` 自动被收录 | astro:content | 被 `index` / `[slug]` 通过 `getCollection` 读取；schema 校验 frontmatter |
| **站点常量** | `src/consts.ts` | 导出 `SITE`（title / short / tagline / description / author / email / social）与 `NAV` | 无 | 被 Base / Header / Footer / Hero / PostCard / about 广泛引用 |
| **设计系统** | `src/styles/global.css` | `:root` 设计 token（颜色 / 渐变 / 圆角 / 字体 / `--maxw`）+ 全局 / 组件 / 响应式样式 | 无 | 被 Base 全局引入，全站生效 |

### 3.4 frontmatter Schema（`src/content/config.ts`）

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `title` | `string` | （必填） | 文章标题，显示在卡片与详情页 |
| `description` | `string` | （必填） | 一句话简介，显示在卡片、详情页与 SEO |
| `date` | `date`（`z.coerce.date()`） | （必填） | 发布日期，用于排序 |
| `tags` | `string[]` | `[]` | 标签，卡片展示前 3 个 |
| `category` | `string` | `"未分类"` | 分类，用于首页右侧栏筛选 |
| `emoji` | `string` | `"🚀"` | 无封面时的卡片占位图 |
| `cover` | `string?` | 可选 | 封面图路径（放 `public/` 下） |
| `author` | `string` | `"你的名字"` | 作者，详情页展示 |
| `draft` | `boolean` | `false` | `true` 时不发布（不进列表与详情） |

> `title` / `description` / `date` 三者齐全即被收录；`draft: true` 可临时隐藏某篇。

## 4. 数据流程

```
[输入]  作者写 Markdown
        src/content/posts/*.md  (frontmatter: title / description / date / category / tags / ...)
            │
            ▼
[处理]  npm run build  (astro build)
   1) Content Collections 读取 + Zod 校验 frontmatter
   2) index.astro → getCollection("posts", !draft) 排序 + 统计分类计数
   3) posts/[slug].astro → getStaticPaths 按每个 .md 生成 /posts/<slug>
   4) Markdown → HTML（Shiki 代码高亮，主题 github-light）
   5) 注入 data-*（卡片）/ headings（TOC）/ 分类计数（侧栏）
            │
            ▼
[存储]  dist/  (纯静态产物)
   ├─ index.html, about/index.html, 404.html, posts/<slug>/index.html
   ├─ _astro/*.css, *.js
   └─ pagefind/  ← postbuild：Pagefind 扫描 dist 给文章正文建全文索引
            │
            ▼
[输出]  托管平台发布 dist/  (Netlify / Cloudflare Pages / Vercel / GitHub Pages)
   └─ 浏览器访问
        • 首页搜索框 → Pagefind.search(term) → 命中 URL → 过滤卡片（降级：data-* 文本匹配）
        • 分类按钮   → activeCat 过滤卡片（与搜索叠加）
        • TOC        → IntersectionObserver 高亮当前章节
        • 响应式     → CSS 媒体查询按视口切换列数 / 导航 / 侧栏位置
```

**关键点**
- **源 Markdown 即「数据库」**：无运行时 DB；内容 = 文件。
- **内容同步 = 重新构建**：每次 `git push` 触发托管平台重新 `npm run build`，即完成「文章更新」。
- **搜索索引是构建产物**：`postbuild` 在 `astro build` 之后对 `dist/` 跑 Pagefind，因此**本地 `npm run dev` 搜索会降级**（索引尚未生成），属预期行为。

## 5. 配置说明（关键项）

| 配置项 | 文件 / 位置 | 作用 |
| --- | --- | --- |
| `site` / `base` | `astro.config.mjs` | 站点绝对地址与子路径基；可用环境变量 `ASTRO_SITE` / `ASTRO_BASE` 覆盖（多平台 / 子路径部署） |
| `markdown.shikiConfig.theme` | `astro.config.mjs` | 代码块高亮主题（现 `github-light`）；`wrap: true` 允许长代码换行 |
| `posts` schema | `src/content/config.ts` | 文章 frontmatter 校验；`category` 默认「未分类」、`draft` 默认 false |
| `--maxw: min(1480px, 95vw)` | `src/styles/global.css` `:root` | 容器最大宽度，**流体自适应**（常规屏 95vw，超宽屏封顶 1480px）；文章详情等页仍受此限以保护阅读行宽 |
| `.home-container { max-width: 100% }` | `global.css` `@media (min-width: 1200px)`（仅首页） | 首页容器在桌面端**放开宽度上限**，4 列随视口宽度自适应；由 `index.astro` 给首页容器加 `home-container` 类触发 |
| 4 列规则 | `global.css` `@media (min-width: 1200px)` | 桌面 4 列 `repeat(4, minmax(0, 1fr))`（列内均分父宽、**不约束总宽**）；平板 2 列、移动（≤720px）1 列 |
| 颜色 / 渐变 token | `global.css` `:root`（`--blue / --violet / --cyan / --magenta` 等） | 改配色集中在此 |
| `postbuild` | `package.json` | `node node_modules/pagefind/lib/runner/bin.cjs --site dist` —— 构建后自动生成全文搜索索引 |
| `build` / `publish` / `NODE_VERSION` | `netlify.toml` | Netlify 构建命令 `npm run build`、发布目录 `dist`、Node 22（Cloudflare Pages / Vercel 等价，无需此文件） |
| GitHub Pages 工作流 | `.github/workflows/deploy.yml` | push `main` 即经 `actions/deploy-pages` 发布到 GitHub Pages；通过 `ASTRO_SITE` / `ASTRO_BASE=/blog/` 环境变量注入子路径，站内链接 / 资源 / Pagefind 索引均自动适配 `/blog/` |
| `import.meta.env.BASE_URL` | 各组件（`Base/Header/Hero/PostCard/[slug]/404/index`） | 所有内部链接、favicon、封面图、Pagefind 索引路径统一用 `BASE_URL` 拼接，使同一套代码在「本地根路径」与「GH Pages 子路径 /blog/」下都正确 |
| `SITE` / `NAV` | `src/consts.ts` | 站名 / 作者 / 社交 / 导航文案，集中维护 |
| 路径别名 `@/*` | `tsconfig.json` | `src/*` 简写导入 |

## 附：技术约束与已知坑（实现层）

1. **Pagefind 在 Windows 的安装**：原生包安装时 trash 报错导致 `.bin` 软链缺失，故 `postbuild` 直接调用 `node node_modules/pagefind/lib/runner/bin.cjs`，跨平台可用。
2. **Pagefind 动态加载路径**：必须用变量 `const pfUrl = "/pagefind/pagefind.js"; await import(/* @vite-ignore */ pfUrl)`，不能用字面量 `import("/pagefind/pagefind.js")`，否则 Astro 构建时 Rollup 会静态解析报错。
3. **中文 / 空格路径**：项目路径含中文与空格，命令行需用引号包裹路径（如 `"D:/OneDrive/桌面/WorkBuddy/blog"`）。
