# 霓光志 · 操作文档

> 面向运维与日常使用：如何准备环境、启动运行、使用各项功能，以及排查常见问题。
> 配套架构见 `docs/架构文档.md`。

## 1. 环境准备

### 1.1 运行环境要求

| 依赖 | 版本要求 | 说明 |
| --- | --- | --- |
| Node.js | **≥ 18.20** 或 **≥ 20**（推荐 22） | 构建与开发依赖；Netlify 侧已锁 `NODE_VERSION=22` |
| 包管理器 | npm（随 Node 自带） | 项目脚本基于 npm |
| Git | 任意较新版本 | 用于内容同步（push 触发部署） |
| 浏览器 | 现代浏览器 | 访问预览与线上站点 |

> 无需数据库、无需后端服务；纯静态站点。

### 1.2 获取代码与安装依赖

```bash
# 1) 进入项目目录（注意路径含中文/空格，命令行需用引号包裹）
cd "D:/OneDrive/桌面/WorkBuddy/blog"

# 2) 安装依赖（含 astro 与 pagefind）
npm install

# 3) 验证安装结果
node -e "console.log(require('./package.json').dependencies, require('./package.json').devDependencies)"
```

安装成功标志：`node_modules/astro` 与 `node_modules/pagefind` 均存在。

## 2. 启动与运行

### 2.1 常用命令

| 命令 | 作用 | 适用场景 |
| --- | --- | --- |
| `npm run dev` | 启动开发服务器（默认 http://localhost:4321），带热更新 | 本地写作 / 调试 |
| `npm run build` | 构建静态站点 → `dist/`，并自动跑 `postbuild` 生成 Pagefind 索引 | 发布前 / CI |
| `npm run preview` | 预览**构建产物** `dist/`（默认 http://localhost:4321） | 验证构建结果 / 测试搜索 |
| `npm run astro` | 直接调用 Astro CLI（如 `npm run astro -- check`） | 进阶维护 |

> `npm run build` 会自动触发 `postbuild`：`node node_modules/pagefind/lib/runner/bin.cjs --site dist`，为 `dist/` 生成全文搜索索引。

### 2.2 运行参数 / 环境变量

| 变量 | 作用 | 示例 |
| --- | --- | --- |
| `ASTRO_SITE` | 覆盖 `astro.config.mjs` 中的 `site`（站点绝对地址，影响 sitemap / 绝对链接） | `ASTRO_SITE=https://your.netlify.app npm run build` |
| `ASTRO_BASE` | 覆盖 `base`（子路径部署，如挂在 `/blog` 下） | `ASTRO_BASE=/blog npm run build` |
| `NODE_VERSION` | Netlify 构建环境 Node 版本（写在 `netlify.toml`） | 已设为 `22` |

### 2.3 各功能使用方法

#### 2.3.1 写 / 更新一篇文章
1. 在 `src/content/posts/` 新建 `xxx.md`，填写 frontmatter（必填 `title` / `description` / `date`）：

   ```md
   ---
   title: "文章标题"
   description: "一句话简介"
   date: 2026-07-12
   category: "AI 应用"   # 首页右侧栏按此筛选
   tags: ["大模型", "教程"]
   emoji: "🚀"
   draft: false          # true 则不发布
   ---

   正文用 Markdown 写……
   ```
2. 本地预览：`npm run dev` → 打开 http://localhost:4321。
3. 提交并推送：

   ```bash
   git add .
   git commit -m "新增：xxx"
   git push
   ```
4. 托管平台检测到 push → 自动 `build` → 发布。**编辑已有 `.md` 同样会自动更新。**

#### 2.3.2 站内搜索（首页右侧栏）
- 在搜索框输入关键词（含**文章正文**中的词），卡片实时过滤，顶部显示结果数。
- 若构建产物中 Pagefind 索引存在（即 `dist/pagefind/pagefind.js` 可访问），搜索命中**正文**；否则自动降级为「标题/描述/标签/分类」文本匹配。

#### 2.3.3 分类筛选（首页右侧栏）
- 点击分类名 → 仅显示该分类文章；点击「全部」重置。
- 分类筛选**可与搜索叠加**（先选分类再输入关键词）。

#### 2.3.4 文章目录（详情页）
- 进入任意文章详情页，右侧自动显示 h2/h3 目录；滚动时当前章节高亮。

#### 2.3.5 改文案 / 配色
- **站名 / 作者 / 导航 / 社交**：编辑 `src/consts.ts`。
- **配色与视觉**：编辑 `src/styles/global.css` 顶部 `:root` 变量。

### 2.4 部署（基于 GitHub 仓库同步）

文章通过 GitHub 仓库同步：写 `.md` → `git push` → 平台自动构建 `dist/` 并发布。

- **方式 A：Netlify（默认，已附 `netlify.toml`）**
  1. Netlify → “Add new site” → 关联 GitHub 仓库。
  2. 构建命令 `npm run build`，发布目录 `dist`（已写在 `netlify.toml`）。
  3. 之后每次 `git push` 到 `main` 自动部署。
- **方式 B：Cloudflare Pages**：关联仓库，构建命令 `npm run build`，输出目录 `dist`，框架预设选 Astro。
- **方式 C：Vercel**：关联仓库，框架预设选 Astro（构建 `npm run build`，输出 `dist`），无需额外配置。

> 三种方式都**不需要任何 Token**；区别只是构建发布方，内容同步仍基于 GitHub 仓库。

## 3. 常见问题排查

### Q1. `npm install` 时 pagefind 报错 / 缺失 `.bin` 软链（Windows）
**现象**：安装过程报 trash 相关错误，`node_modules/.bin/pagefind` 不存在，构建时 `postbuild` 报找不到命令。
**原因**：Windows 下 pagefind 原生二进制安装时 trash 删除失败，导致 npm 未创建 bin 软链。
**解决**：项目已将 `postbuild` 改为直接调用 runner，跨平台可用，无需 `.bin`：
```json
"postbuild": "node node_modules/pagefind/lib/runner/bin.cjs --site dist"
```
若仍异常，重跑 `npm install`（保留 `--no-audit --no-fund`）；确认 `node_modules/pagefind` 与 `node_modules/@pagefind/windows-x64/bin/` 存在即可。

### Q2. 构建报错：无法解析 `/pagefind/pagefind.js`（Rollup）
**现象**：`astro build` 失败，提示 `Failed to resolve import "/pagefind/pagefind.js"`。
**原因**：用字面量 `import("/pagefind/pagefind.js")` 会被 Rollup 在构建时静态解析，但该文件是构建后才生成的。
**解决**：使用变量路径 + `@vite-ignore`：
```js
const pfUrl = "/pagefind/pagefind.js";
pf = await import(/* @vite-ignore */ pfUrl);
```

### Q3. 搜索没反应 / 搜不到正文
**现象**：首页搜索框输入无过滤，或只能匹配标题。
**原因**：`dist/pagefind/` 索引未生成——多为**直接用 `npm run dev` 预览**（未构建，索引不存在），此时走的是降级文本匹配。
**解决**：
- 验证索引存在：`curl -s -o /dev/null -w "%{http_code}" http://localhost:4321/pagefind/pagefind.js` 应返回 200。
- 先 `npm run build` 再 `npm run preview`，在预览态测试全文搜索。
- 确保 `postbuild` 已执行（构建日志应出现 Pagefind 索引生成信息）。

### Q4. 部署后页面空白 / 资源 404（尤其是子路径）
**现象**：本地正常，部署后样式或页面 404。
**解决**：
- 若有子路径部署，设置 `ASTRO_BASE=/你的子路径` 后重新构建。
- 确认托管平台的「发布目录」为 `dist`、「构建命令」为 `npm run build`。
- 确认 Node 版本 ≥ 18.20（Netlify 已锁 22）。

### Q5. 新增 `.md` 不出现在首页 / 详情页打不开
**现象**：放了文章但首页没有，或 `/posts/xxx` 404。
**排查**：
- frontmatter 是否缺 `title` / `description` / `date`（三者必填，缺则 schema 校验失败，该篇被忽略）。
- 是否设了 `draft: true`（草稿不发布）。
- 是否未重新 `build` / 未 `git push` 触发部署。
- 日期格式是否合法（`z.coerce.date()` 解析失败会导致忽略）。

### Q6. 中文 / 空格路径导致命令失败
**现象**：`cd` 或脚本报错路径找不到。
**解决**：命令行用引号包裹含中文或空格的路径，例如 `"D:/OneDrive/桌面/WorkBuddy/blog"`。

### Q7. 字体未加载 / 显示异常
**现象**：标题字体不是 Space Grotesk 等。
**原因**：Google Fonts 通过 `<link>` 在线加载，需联网；离线或被墙时回退系统字体（不影响功能）。
**解决**：确认构建/访问环境可访问 `fonts.googleapis.com`；或改为自托管字体。

### Q8. 构建成功但 `dist/` 不含某页面
**现象**：页面缺失。
**排查**：检查是否缺少对应 `src/pages/*.astro`；文章详情由 `getStaticPaths` 自动生成，依赖 `src/content/posts/` 下的 `.md`。
